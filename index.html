O Ponto de Partida

O que é Portugol?

Antes de aprender qualquer linguagem de programação, você precisa pensar como um computador. O Portugol te ensina exatamente isso.

🤖
AlgoMaster
Portugol é um pseudocódigo em português. Ele não roda em nenhum computador real — e isso é proposital! O objetivo não é executar o código, mas sim treinar o raciocínio lógico sem se preocupar com a sintaxe de uma linguagem específica.
🎓
Prof. Guanabara diria:
Quando você escreve em Portugol, você está descrevendo a solução, não programando. É como fazer um rascunho antes de pintar. Depois que a lógica estiver certa, traduzir para Python, JavaScript ou Java é a parte fácil!
🤖
AlgoMaster
No Portugol, usamos palavras como algoritmo, inicio, fim, escreva, leia, se, enquanto — tudo em português! Isso elimina a barreira do idioma e deixa você focar no que importa: a lógica.
💡
Por que aprender Portugol primeiro? Alunos que aprendem lógica com pseudocódigo antes de uma linguagem real consolidam o aprendizado muito mais rápido. Você não luta contra a sintaxe — você pensa na solução.
📝
Pseudocódigo
Texto que imita código mas é legível por humanos. Não tem regras rígidas de sintaxe — só lógica. Portugol é um pseudocódigo padronizado para o português.
🔄
Algoritmo
Uma sequência finita de passos bem definidos para resolver um problema. Todo programa é um algoritmo — Portugol te ensina a criar bons algoritmos.
🧠
Lógica de Programação
A capacidade de decompor um problema em passos lógicos e ordená-los corretamente. É a habilidade mais importante de um programador — e transferível para qualquer linguagem.
⚙️
Visualg 3.0
Ferramenta gratuita onde você pode executar Portugol de verdade! O Visualg é o mais usado no Brasil e perfeito para iniciantes executarem seus algoritmos.

Fluxograma

Antes de escrever qualquer linha de Portugol, desenhe o fluxograma. Ele é o mapa do seu algoritmo — você pensa com o lápis antes de codificar.

🎓
Prof. Guanabara diria:
Todo algoritmo que você escrever à mão ou no Visualg, comece pelo fluxograma. Ele força você a pensar em todos os caminhos possíveis antes de codificar. Se o fluxograma estiver certo, o Portugol será só uma tradução!

Processo (cálculo, atribuição)

Exemplo 1 — Verificar Aprovação
🤖
AlgoMaster
Leia o fluxograma de cima para baixo, seguindo as setas. Nos losangos de decisão, o caminho bifurca em Sim ou Não. Ao lado, veja como cada forma se traduz diretamente em Portugol.
Portugol correspondente
Cada forma do fluxograma vira uma linha de Portugol. Perceba como os dois contam a mesma história — um visualmente, o outro em texto.
Portugol — do fluxo ao código 
algoritmo "Aprovacao"
var
   nota : real

inicio
   // 🔷 INÍCIO do fluxo

   // ▱ Entrada de dados
   leia(nota)

   // ◇ 1ª Decisão: nota >= 7?
   se (nota >= 7.0) entao
      escreva("✅ Aprovado")
   senao
      // ◇ 2ª Decisão: nota >= 5?
      se (nota >= 5.0) entao
         escreva("⚠️ Recuperação")
      senao
         escreva("❌ Reprovado")
      fimse
   fimse

   // 🔷 FIM do fluxo
fimalgoritmo
Exemplo 2 — Somar Números com Laço
🤖
AlgoMaster
Laços no fluxograma criam uma seta que volta para cima — é isso que define o loop visualmente. Enquanto a condição for verdadeira, o fluxo fica girando naquele ciclo. Quando for falsa, ele escapa e segue em frente.
Portugol correspondente
A seta que volta para cima no fluxograma vira o laço enquanto no Portugol. O losango de decisão é a condição do laço.
Portugol — laço enquanto 
algoritmo "SomaComLaco"
var
   soma, i : inteiro

inicio
   // ▭ Processo: inicialização
   soma <- 0
   i    <- 1

   // ◇ Decisão: condição do laço
   enquanto (i <= 5) faca
      // ▭ Processo: acumula soma
      soma <- soma + i
      // ▭ Processo: incrementa i
      i    <- i + 1
      // ↑ seta volta para o ◇
   fimenquanto

   // ▱ Saída de dados
   escreva("Soma: ", soma) // → 15
fimalgoritmo

Fundamentos

Conceitos Essenciais

Os blocos de construção de qualquer algoritmo. Entenda cada um antes de avançar.

📦 Variáveis e Tipos
🤖
AlgoMaster
Uma variável é uma caixa com nome que guarda um valor na memória. Em Portugol, você precisa declarar a variável antes de usar, dizendo qual é o tipo do dado que ela vai guardar.
Portugol — Variáveis 
algoritmo "ExemploVariaveis"
var
   // ① Declaração de variáveis com seus tipos
   nome     : caractere   // guarda texto
   idade    : inteiro     // guarda número inteiro
   altura   : real        // guarda número decimal
   aprovado : logico      // guarda verdadeiro/falso

inicio
   // ② Atribuindo valores (sinal <-)
   nome     <- "Ana"
   idade    <- 20
   altura   <- 1.68
   aprovado <- verdadeiro

   // ③ Exibindo os valores
   escreva("Nome: ", nome)
   escreva("Idade: ", idade)
   escreva("Aprovado: ", aprovado)
fimalgoritmo
①
var é onde você lista todas as variáveis. Cada uma tem um nome e um tipo separados por dois-pontos.
②
O símbolo <- é a atribuição. Lê-se "recebe". Ex: nome <- "Ana" significa: a variável nome recebe o valor Ana.
③
escreva() exibe algo na tela. É o famoso "print" do Portugol.
✅
Os 4 tipos de dados do Portugol: inteiro — números sem vírgula (1, 42, -7)  ·  real — com vírgula (3.14, 1.68)  ·  caractere — texto ("Olá")  ·  logico — verdadeiro ou falso
🔀 Estruturas de Decisão
💻
Conceito
Decisões são o coração de qualquer algoritmo. Sem elas, o programa faz a mesma coisa sempre. Com elas, ele reage a diferentes situações. Usamos se ... entao ... senao em Portugol.
Portugol — Se / Então / Senão 
algoritmo "VerificaAprovacao"
var
   nota : real

inicio
   escreva("Digite sua nota: ")
   leia(nota)   // ① captura o que o usuário digita

   se (nota >= 7.0) entao           // ②
      escreva("✅ Aprovado! Nota: ", nota)
   senao
      se (nota >= 5.0) entao         // ③ se aninhado
         escreva("⚠️ Recuperação: ", nota)
      senao
         escreva("❌ Reprovado: ", nota)
      fimse
   fimse
fimalgoritmo
①
leia() é o oposto de escreva. Ele espera o usuário digitar e guarda o valor na variável.
②
Todo se...entao precisa terminar com fimse. É como fechar um parêntese — obrigatório!
③
Você pode colocar um se dentro de outro — isso é se aninhado. Use com moderação para não perder a legibilidade.
🔁 Estruturas de Repetição
🤖
AlgoMaster
Computadores são ótimos em fazer a mesma coisa muitas vezes sem reclamar. As estruturas de repetição — chamadas de laços ou loops — são o que permite isso. Portugol tem três: enquanto, para e repita...ate.
Portugol — Os 3 Laços 
algoritmo "ExemploLacos"
var
   i, soma : inteiro

inicio
   // ① PARA — quando você sabe quantas vezes vai repetir
   para i <- 1 ate 5 faca
      escreva("Contando: ", i)
   fimpara

   // ② ENQUANTO — testa a condição ANTES de executar
   soma <- 0
   i    <- 1
   enquanto (i <= 10) faca
      soma <- soma + i
      i    <- i + 1
   fimenquanto
   escreva("Soma de 1 a 10: ", soma)  // → 55

   // ③ REPITA...ATÉ — executa pelo menos 1 vez, testa DEPOIS
   repita
      escreva("Digite um número positivo: ")
      leia(i)
   ate (i > 0)
fimalgoritmo
①
para...ate...faca — ideal quando você sabe o número de repetições. Equivale ao for de outras linguagens.
②
enquanto...faca — testa a condição antes. Se já for falsa no início, o bloco nunca executa.
③
repita...ate — testa a condição depois. O bloco executa pelo menos uma vez. Ótimo para validar entradas do usuário.
⚠️
Cuidado com o Loop Infinito! Se a condição do enquanto nunca se tornar falsa, o programa trava para sempre. Sempre certifique-se de que a variável de controle está sendo atualizada dentro do laço!

Algoritmos em Portugol

Exemplos comentados dos algoritmos mais importantes — primeiro em Portugol para entender a lógica, depois em Python como referência de linguagem real.

🔀 Ordenação — Bubble Sort
🧩
Conceito
O Bubble Sort compara dois elementos vizinhos e os troca se estiverem fora de ordem. Ele faz isso várias vezes até tudo estar certo, e o maior valor vai "subindo" como uma bolha na água até o final da lista.
Portugol — Bubble Sort · O(n²) 
algoritmo "BubbleSort"
var
   vet       : vetor[1..5] de inteiro
   i, j, aux : inteiro

inicio
   // Preenchendo o vetor com entradas do usuário
   para i <- 1 ate 5 faca
      escreva("Digite o ", i, "º número: ")
      leia(vet[i])
   fimpara

   // ① Laço externo: número de passagens
   para i <- 1 ate 4 faca
      // ② Laço interno: compara pares vizinhos
      para j <- 1 ate 5 - i faca
         se (vet[j] > vet[j+1]) entao
            // ③ Troca usando variável auxiliar
            aux        <- vet[j]
            vet[j]     <- vet[j+1]
            vet[j+1]   <- aux
         fimse
      fimpara
   fimpara

   escreva("Vetor ordenado: ")
   para i <- 1 ate 5 faca
      escreva(vet[i], " ")
   fimpara
fimalgoritmo
Python — mesma lógica, outra sintaxe 🔍 Compare com o Portugol acima 
# A lógica é IDÊNTICA ao Portugol — só a sintaxe muda!
def bubble_sort(vet):
    n = len(vet)
    for i in range(n):
        for j in range(n-i-1):
            if vet[j] > vet[j+1]:
                # Python troca sem variável auxiliar!
                vet[j], vet[j+1] = vet[j+1], vet[j]
    return vet

print(bubble_sort([5, 2, 8, 1, 9]))  # → [1, 2, 5, 8, 9]
🔍 Busca Binária
🤖
AlgoMaster
Imagine procurar uma palavra num dicionário. Você não lê página por página — você abre no meio, vê se está antes ou depois, e divide de novo. Isso é a Busca Binária: muito mais rápida que procurar um por um, mas só funciona em listas já ordenadas!
Portugol — Busca Binária · O(log n) 
algoritmo "BuscaBinaria"
var
   vet            : vetor[1..7] de inteiro
   alvo           : inteiro
   esq, dir, meio : inteiro
   achou          : logico

inicio
   // Vetor JÁ ordenado — obrigatório!
   vet[1]<-2  vet[2]<-5  vet[3]<-9  vet[4]<-14
   vet[5]<-21 vet[6]<-37 vet[7]<-42

   escreva("Qual número procurar? ")
   leia(alvo)

   esq   <- 1
   dir   <- 7
   achou <- falso

   enquanto (esq <= dir) e (nao achou) faca
      meio <- (esq + dir) div 2    // ① ponto do meio

      se (vet[meio] = alvo) entao
         achou <- verdadeiro          // ② encontrou!
      senao
         se (vet[meio] < alvo) entao
            esq <- meio + 1           // ③ busca à direita
         senao
            dir <- meio - 1           // ③ busca à esquerda
         fimse
      fimse
   fimenquanto

   se (achou) entao
      escreva("✅ Encontrado na posição ", meio)
   senao
      escreva("❌ Número não encontrado")
   fimse
fimalgoritmo
Python — Busca Binária 
def busca_binaria(vet, alvo):
    esq, dir = 0, len(vet) - 1
    while esq <= dir:
        meio = (esq + dir) // 2
        if   vet[meio] == alvo: return meio
        elif vet[meio] <  alvo: esq = meio + 1
        else:                    dir = meio - 1
    return -1

lista = [2, 5, 9, 14, 21, 37, 42]
print(busca_binaria(lista, 21))  # → 4 (índice)
🔍
Por que Busca Binária é tão rápida? Numa lista de 1.000 elementos, a busca linear pode precisar de até 1.000 comparações. A busca binária precisa de no máximo 10 (log₂ 1000 ≈ 10). Numa lista de 1 bilhão? Apenas 30 comparações!

Armadilhas Clássicas

Erros Comuns em Portugol

Todo iniciante comete esses erros. Conhecê-los antes de encontrá-los poupa horas de frustração no Visualg.

🤖
AlgoMaster
O Visualg vai te dar mensagens de erro nem sempre claras. Conhecer os bugs mais comuns de antemão é como ter um mapa de minas — você passa por cima sem explodir. Veja o errado ao lado do certo e grave a diferença.
Erro 1 — Atribuição com = em vez de <-
❌ Errado
inicio
   idade = 20   // ← isso não existe!
   escreva(idade)
fimalgoritmo
Em Portugol, = é só para comparação (dentro de um se). Para atribuir valor, sempre use <-.
✅ Certo
inicio
   idade <- 20   // "idade recebe 20"
   escreva(idade)
fimalgoritmo
Leia sempre como "recebe": idade <- 20 = "idade recebe 20". Isso reforça o conceito de memória.
Erro 2 — Esquecer o fimse
❌ Errado
se (nota >= 7) entao
   escreva("Aprovado")
// esqueceu o fimse!
escreva("Fim")
fimalgoritmo
Sem o fimse, o Visualg não sabe onde o bloco condicional termina e gera erro de sintaxe.
✅ Certo
se (nota >= 7) entao
   escreva("Aprovado")
fimse   // fecha o bloco se
escreva("Fim")
fimalgoritmo
Cada se precisa de um fimse. Cada enquanto precisa de fimenquanto. Cada para precisa de fimpara. É sempre um par.
Erro 3 — Usar variável sem declarar
❌ Errado
algoritmo "Teste"
// bloco var ausente!
inicio
   soma <- 10   // erro: soma não existe
   escreva(soma)
fimalgoritmo
Portugol é uma linguagem de tipagem estática. Toda variável deve ser declarada no bloco var com seu tipo antes de ser usada.
✅ Certo
algoritmo "Teste"
var
   soma : inteiro   // declarada aqui
inicio
   soma <- 10
   escreva(soma)   // → 10
fimalgoritmo
A seção var fica sempre entre o nome do algoritmo e o inicio. Pense nela como a "lista de caixas" que você vai usar.
Erro 4 — Loop infinito por não incrementar o contador
❌ Errado (loop infinito!)
var
   i : inteiro
inicio
   i <- 1
   enquanto (i <= 5) faca
      escreva(i)
      // i nunca muda → trava!
   fimenquanto
fimalgoritmo
Se a condição nunca se tornar falsa, o laço roda para sempre. O Visualg trava e você precisa forçar parada.
✅ Certo
var
   i : inteiro
inicio
   i <- 1
   enquanto (i <= 5) faca
      escreva(i)
      i <- i + 1   // incrementa!
   fimenquanto
fimalgoritmo
Sempre verifique: algo dentro do laço deve aproximar a condição do falso. Normalmente é um incremento ou decremento.
Erro 5 — Confundir leia() e escreva()
❌ Errado
var
   nome : caractere
inicio
   // invertido!
   leia("Qual seu nome? ")
   escreva(nome)
fimalgoritmo
leia() recebe valores do teclado — não aceita texto fixo. escreva() exibe na tela — não lê entrada.
✅ Certo
var
   nome : caractere
inicio
   escreva("Qual seu nome? ")
   leia(nome)   // recebe do teclado
   escreva("Olá, ", nome)
fimalgoritmo
O padrão é sempre: escreva(pergunta) → leia(variável). Primeiro você orienta o usuário, depois coleta a resposta dele.
💡
Dica de ouro: Sempre que o Visualg travar ou der erro, cheque nessa ordem: (1) todas as variáveis estão declaradas? (2) todos os se/enquanto/para têm seu fecho? (3) o laço tem incremento? Resolve 90% dos problemas.

Calculadora de Média
Iniciante
Crie um algoritmo em Portugol que:
• Leia 3 notas de um aluno (variáveis n1, n2, n3 do tipo real)
• Calcule a média aritmética
• Exiba a média e informe se o aluno está Aprovado (média ≥ 7), em Recuperação (média ≥ 5) ou Reprovado
Dica: A média é (n1 + n2 + n3) / 3. Declare uma variável media : real para guardar o resultado. Depois use se ... senao ... se ... fimse aninhado para as 3 condições.
Gabarito — Calculadora de Média 
algoritmo "MediaAluno"
var
   n1, n2, n3, media : real

inicio
   escreva("Digite a 1ª nota: ")  leia(n1)
   escreva("Digite a 2ª nota: ")  leia(n2)
   escreva("Digite a 3ª nota: ")  leia(n3)

   media <- (n1 + n2 + n3) / 3
   escreva("Média: ", media)

   se (media >= 7) entao
      escreva("✅ Aprovado")
   senao
      se (media >= 5) entao
         escreva("⚠️ Recuperação")
      senao
         escreva("❌ Reprovado")
      fimse
   fimse
fimalgoritmo

Tabuada Personalizada

Crie um algoritmo que:

Contador de Pares e Ímpares
Intermediário
Crie um algoritmo que:
• Leia 10 números inteiros digitados pelo usuário (use um laço)
• Conte quantos são pares e quantos são ímpares
• Exiba os totais ao final
Dica de contexto: um número é par se o resto da divisão por 2 for zero. Em Portugol, o operador de resto é mod.
Dica: Declare pares, impares, num, i : inteiro. Inicialize pares e impares com 0. No laço para i <- 1 ate 10, leia o número e teste: se (num mod 2 = 0) incremente pares, senão incremente ímpares.
Gabarito — Pares e Ímpares 
algoritmo "ParesImpares"
var
   num, i, pares, impares : inteiro

inicio
   pares   <- 0
   impares <- 0

   para i <- 1 ate 10 faca
      escreva("Digite o ", i, "º número: ")
      leia(num)

      se (num mod 2 = 0) entao
         pares <- pares + 1
      senao
         impares <- impares + 1
      fimse
   fimpara

   escreval("Pares:   ", pares)
   escreval("Ímpares: ", impares)
fimalgoritmo

Maior e Menor de uma Sequência
Intermediário
Crie um algoritmo que:
• Leia 5 números reais
• Encontre e exiba o maior e o menor valor digitado
Desafio extra: tente também calcular a soma e a média dos 5 números.
Dica: Leia o primeiro número antes do laço e use-o para inicializar tanto maior quanto menor. No laço, leia os 4 restantes comparando com se (num > maior) e se (num < menor).
Gabarito — Maior e Menor 
algoritmo "MaiorMenor"
var
   num, maior, menor, soma : real
   i : inteiro

inicio
   escreva("1º número: ")
   leia(num)
   maior <- num   // assume o 1º como base
   menor <- num
   soma  <- num

   para i <- 2 ate 5 faca
      escreva(i, "º número: ")
      leia(num)
      soma <- soma + num

      se (num > maior) entao
         maior <- num
      fimse
      se (num < menor) entao
         menor <- num
      fimse
   fimpara

   escreval("Maior: ", maior)
   escreval("Menor: ", menor)
   escreval("Média: ", soma / 5)
fimalgoritmo

Verificador de Número Primo
Avançado
Crie um algoritmo que:
• Leia um número inteiro positivo
• Determine se ele é primo ou não
• Exiba a resposta com uma mensagem clara
Lembre: um número primo só é divisível por 1 e por ele mesmo. 1 não é primo.
Dica: Use uma variável primo : logico iniciada como verdadeiro. Faça um laço de i <- 2 ate num-1: se num mod i = 0, defina primo <- falso. Ao final, teste se (primo e num > 1).
Gabarito — Número Primo 
algoritmo "NumeroPrimo"
var
   num, i : inteiro
   primo   : logico

inicio
   escreva("Digite um número: ")
   leia(num)

   primo <- verdadeiro

   se (num <= 1) entao
      primo <- falso
   senao
      para i <- 2 ate num - 1 faca
         se (num mod i = 0) entao
            primo <- falso
         fimse
      fimpara
   fimse

   se (primo) entao
      escreval(num, " é primo ✅")
   senao
      escreval(num, " não é primo ❌")

Notação Big O

Depois de aprender a lógica em Portugol, o próximo nível é entender quão eficiente é o seu algoritmo.

🤖
AlgoMaster
Big O responde à pergunta: "Se eu dobrar o tamanho da entrada, quanto mais lento fica?" Um O(n) dobra o tempo. Um O(n²) fica 4× mais lento. Um O(log n) quase não muda. Isso importa muito no mundo real.

Livros Recomendados

Do iniciante absoluto ao avançado — os melhores livros para cada etapa da jornada.

📗
Iniciante
Entendendo Algoritmos
Aditya Y. Bhargava
Introdução visual com centenas de ilustrações. Perfeito para quem está começando — aborda desde busca binária até programação dinâmica de forma leve e acessível.
📙
Iniciante / Médio
Lógica de Programação e Algoritmos
Edécio Fernando Iepsen
Focado no mercado brasileiro, usa Portugol como base. Excelente para quem quer solidificar a lógica antes de partir para uma linguagem real. Muito adotado em faculdades.
📘
Médio / Avançado
Algoritmos: Teoria e Prática (CLRS)
Cormen, Leiserson, Rivest, Stein
A "bíblia" dos algoritmos. Denso e matemático — não é para iniciantes, mas é a referência definitiva da área. Todo programador sério deve conhecer.
📕
Médio
The Algorithm Design Manual
Steven S. Skiena
Abordagem prática com foco em resolver problemas reais. Inclui catálogo de algoritmos e casos de uso do mundo real. Ótima leitura após dominar o básico.
🟣
Entrevistas
Cracking the Coding Interview
Gayle Laakmann McDowell
189 problemas com soluções detalhadas. O guia mais usado para entrevistas técnicas em grandes empresas de tecnologia. Foca em raciocínio algorítmico.
🟡
Iniciante
Algoritmos — Unlocking the Secrets
Thomas H. Cormen
Versão acessível do CLRS escrita pelo próprio Cormen. Apresenta os conceitos sem o formalismo pesado — ponte perfeita entre o básico e o livro definitivo.
🟠
Médio
Estruturas de Dados e Algoritmos com JavaScript
Loiane Groner
Abordagem prática para quem já conhece o básico e quer aprender estruturas de dados usando JavaScript. Cobre listas, pilhas, filas, árvores e grafos com exemplos reais e código limpo.
🔶
Avançado
Programming Pearls
Jon Bentley
Um clássico sobre como pensar em problemas difíceis de forma elegante. Ensina a arte de escolher o algoritmo certo para o problema certo — leitura obrigatória para quem quer escrever código eficiente e bem pensado.
